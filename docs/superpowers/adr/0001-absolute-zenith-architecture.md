# ADR 0001: Фундаментальная Архитектура "Absolute Zenith" (Bevy 0.18.1)

* **Статус**: Accepted
* **Дата**: 2026-05-06
* **Автор**: Savage Fantasy Agent

## Контекст (Context)
Проект "Savage Fantasy" требует высокой производительности, масштабируемости и чистоты кода для совместной разработки человеком и ИИ. Необходимо было выбрать паттерны, которые минимизируют ошибки (runtime panics), обеспечивают максимальный параллелизм ECS и легко проверяются автотестами.

## Решение (Decision)
Внедрена многослойная архитектура, базирующаяся на:
1. **Entity Relations (Bevy 0.18.1)**: Использование типизированных связей (`relationship`) вместо хранения сырых `Entity` ID.
2. **Safe Behavior Switcher (ZST)**: Использование Zero-Sized Types для состояний ИИ и атомарного переключения через `switch_behavior` API.
3. **Plugins 2.0 (Named Commands)**: Публичный API модулей реализован через расширения `Commands` с использованием именованных структур `Command`.
4. **Opaque Newtypes**: Все статистические данные (`Hunger`, `Morale`) инкапсулированы в типы с приватными полями.
5. **Architectural Guards**: 29+ автоматизированных тестов, проверяющих соблюдение правил (отсутствие `unwrap`, использование System Sets и т.д.).

## Обоснование (Rationale)
Это решение обеспечивает "Жесткий потолок" качества. Bevy 0.18.1 предоставляет нативные инструменты для графовых связей и типизированных команд, что делает архитектуру "родной" для движка и максимально эффективной для планировщика задач (scheduler).

## Последствия (Consequences)
* **Положительные**: Почти нулевой риск повреждения стейта (race conditions) в логике. Легкая отладка через профайлеры (благодаря именованным командам). Высокая скорость онбординга новых AI-агентов.
* **Отрицательные**: Более высокий порог входа (необходимо понимать трейты и макросы Bevy). Запрет на использование "быстрых, но грязных" хаков (блокируется гвардейцами).
* **Нейтральные**: Необходимость ведения ADR и соблюдения стандартов коммитов.
