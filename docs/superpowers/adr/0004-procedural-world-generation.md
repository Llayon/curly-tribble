# ADR 0004: Процедурная Генерация Мира (Procedural World Generation)

* **Статус**: Accepted
* **Дата**: 2026-05-09
* **Автор**: Savage Fantasy Agent

## Контекст (Context)
Для создания реиграбельности и живого мира (в духе RimWorld), карта не должна быть статичной. Необходимо генерировать ландшафт, биомы и распределение ресурсов автоматически, используя «зерно» (Seed).

## Решение (Decision)
Внедрена гибридная система процедурной генерации "Bones/Climate/Carving":
1. **The Bones (Voronoi + Fbm)**: Использование диаграмм Вороного для создания структуры макро-рельефа (горные хребты и низины) в сочетании с Fbm шумом для микро-детализации (холмы).
2. **The Climate (Whittaker Biomes)**: Распределение биомов на основе двух независимых карт шума — Температуры и Влажности (матрица Уиттекера). Это позволяет логично размещать болота, пустыни и леса.
3. **The Carving (Stamp System)**: Внедрение системы "штампов" для генерации уникальных пещер и POI, которые бесшовно "вплавляются" в горные массивы.
4. **Seamless Reveal (Dithered Alpha)**: Автоматическое скрытие "крыш" пещер при нахождении юнитов внутри, обеспечивающее тактическую связность без экранов загрузки.
5. **Deterministic Scattering**: Ресурсы (деревья, камни, кусты) спавнятся на основе биомных правил (например, кактусы в пустыне) с использованием `rand::StdRng`.
6. **GLTF Asset Integration**: Использование полноценных 3D сцен Quaternius через `SceneRoot`.

7. **3D Verticality (MAX_HEIGHT: 4.0)**: Визуальное "выдавливание" рельефа на основе карты высот. Каждый тайл имеет координату `y`, соответствующую его `elevation`.
8. **2.5D Navigation (Slope-Aware)**: Система навигации учитывает крутизну склонов. Если разница высот между соседними клетками `> 0.3`, путь блокируется (`COST_BLOCKER`). Это обеспечивает реалистичное движение без усложнения алгоритма A*.
9. **Smooth Terrain Mesh**: Устранение визуальных разрывов («летающих ковров») путем сшивки углов тайлов. Высота каждого угла меша вычисляется как среднее арифметическое высот четырех соседних тайлов.
10. **Global Indexed Mesh & Smooth Shading**: Переход к единому индексированному мешу для всей карты с использованием `compute_normals()`. Это обеспечивает гладкое затенение и высокую производительность (1 Draw Call для всего ландшафта).
11. **Vertex Biome Blending**: Использование цветов вершин (`ATTRIBUTE_COLOR`) для бесшовного смешивания биомов (трава, песок, грязь).
12. **Dedicated Water Layer**: Вынос воды в отдельный прозрачный слой на фиксированной высоте `WATER_LEVEL` с использованием `AlphaMode::Blend`.
13. **Global Roof Mesh**: Объединение крыш пещер в единый плавный индексированный меш (аналогично ландшафту), что устраняет «пиксельные» артефакты и обеспечивает плавное освещение горных вершин.

## Обоснование (Rationale)
Гибридный подход (Voronoi + Noise) решает проблему "однообразного шума", создавая выразительную географию. Система биомов Уиттекера обеспечивает стратегическую глубину, а система штампов позволяет гарантировать наличие интересных локаций для исследования. Использование единых индексированных мешей со сглаженными нормалями для ландшафта и горных вершин превращает дискретную сетку в реалистичный 3D-ландшафт мирового уровня, радикально снижая количество вызовов отрисовки и обеспечивая современное визуальное качество без графических артефактов.

## Последствия (Consequences)
* **Положительные**: Каждый запуск уникален. Навигация автоматически адаптируется к новой карте. Улучшен визуал.
* **Отрицательные**: Увеличение времени компиляции (новые зависимости `noise`, `rand`).
* **Нейтральные**: Все объекты на карте должны учитывать `NavObstacle` для корректной работы Zenith Navigator.
