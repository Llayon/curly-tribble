# ADR 0004: Процедурная Генерация Мира (Procedural World Generation)

* **Статус**: Accepted
* **Дата**: 2026-05-09
* **Автор**: Savage Fantasy Agent

## Контекст (Context)
Для создания реиграбельности и живого мира (в духе RimWorld), карта не должна быть статичной. Необходимо генерировать ландшафт, биомы и распределение ресурсов автоматически, используя «зерно» (Seed).

## Решение (Decision)
Внедрена система процедурной генерации, основанная на:
1. **Fbm Noise (Perlin)**: Использование библиотеки `noise` для создания плавных переходов между типами ландшафта (Трава, Грязь, Вода).
2. **Terrain Hierarchy**: Каждому типу ландшафта присваивается `TerrainType` и соответствующий вес навигации (Grass: 20, Mud: 50, Water: Blocker).
3. **Deterministic Scattering**: Ресурсы (кусты, деревья) спавнятся с использованием `rand::StdRng`, инициализированного тем же сидом, что и шум, но только на валидных тайлах (Grass).
4. **GLTF Asset Integration**: Замена процедурных примитивов (сферы, кубы) на полноценные 3D сцены из пака Quaternius через `SceneRoot`.

## Обоснование (Rationale)
Шум Перлина обеспечивает естественный вид ландшафта («острова» и «озера»). Разделение на фазы (генерация почвы -> рассеивание объектов) позволяет легко настраивать баланс ресурсов без изменения геометрии.

## Последствия (Consequences)
* **Положительные**: Каждый запуск уникален. Навигация автоматически адаптируется к новой карте. Улучшен визуал.
* **Отрицательные**: Увеличение времени компиляции (новые зависимости `noise`, `rand`).
* **Нейтральные**: Все объекты на карте должны учитывать `NavObstacle` для корректной работы Zenith Navigator.
